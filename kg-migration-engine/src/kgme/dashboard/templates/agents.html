One question comes in → a router picks exactly one of these six → one answer comes back. This page exists to make that easy to explain to someone new.

The one thing that matters most

Only As-Is lets an AI write its own database query. The other five run a fixed query a human wrote — the AI only ever turns already-correct results into readable sentences. It cannot change what data comes back.

As-IsThe other 5
Who decides what to query? The AI, per question A human, once, hand-verified
Can the AI influence the data returned? No — two independent safety checks review the query first No — there's no query for it to influence
What can the AI actually do? Write the query and the answer Only phrase the answer

The 6 agents

1 As-Is writes its own query

Answers arbitrary factual questions — "what is X, what uses Y, who authorizes Z" — by writing its own database query on the fly. The only agent that does this, so it's the most flexible, and the only one with extra guardrails (a write-blocking check, plus a database-level read-only lock). Knows about every property on the graph, including ones added by later enrichment (like the SAP catalog data — see agent 3).

"What SAP transactions does business process MM01 use?"

2 Migration-Mapping fixed query

Reports what fraction of a module's transactions have a known migration target, honestly — a module with zero mappings is reported as zero, never hidden. Every mapping is explicitly labeled the analyst's own inferred opinion, never a sourced fact, until a human confirms it.

"How much of the MM module has a confirmed migration target?" → "3 of 12 (25%)."

Known gap (not yet built): doesn't cross-check its own mappings against the SAP Simplification Catalog data yet — even where the two would agree. Planned, not started.

3 GxP-Compliance fixed query

Surfaces two independent risk categories: GxP risk findings (undocumented gaps, unverified guesses, or paths into a GxP-critical process — one specific finding, an automated batch-release step with an unverified data source, is always shown first regardless of severity), and SAP Simplification Catalog findings (nodes the official SAP migration catalog flags as breaking, reported separately, always labeled external/unconfirmed).

"What is the compliance risk with QM:BATCH_RELEASE and LAB_SYSTEM?"

4 Cross-Module Impact fixed query

Finds the one place a Materials-Management chain and an Asset-Accounting chain meet, and reports which single link in that chain is least trustworthy — the one most likely to silently break during migration.

"How do the MM and AM modules connect to each other?"

5 Gaps fixed query

A plain inventory: which documents/nodes are referenced but never actually received, and which relationships are only an inferred guess. Deliberately not a risk narrative — just a list of things to go chase down.

"What documents are referenced in this graph but not in our possession?"

6 Temporal no query at all

The graph has no field tracking a document's age, so this agent always says so plainly, rather than guessing. No database query, no AI call — a fixed, honest answer, because there's genuinely nothing to compute.

"How old is the documentation behind MM01, and is it still valid?"

How a question finds its way to the right agent

One AI call reads the question and picks exactly one of the 6 routes — reasoning about meaning, not matching keywords. It's structurally forced to return one of the 6 exact names, so it can't return something malformed or made up.

Question sounds like...Actually routes to
"What compliance risks exist?"Compliance (risk framing)
"What documents are missing?"Gaps (plain inventory, same underlying data)
"Any SAP catalog findings for module X?"Compliance (even without risk language — a broad catalog survey)
"What does the catalog say about one specific transaction?"As-Is (a targeted lookup, not a survey)

Every fact any agent states is tagged documented (a real source says so) or inferred/gap (a guess, or a known hole) — and an agent never states one as fact without the other. If there's no data to answer something, every agent says so plainly instead of making something up.
